## C. 에피토프·설계 파일럿 (착수 게이트)

**착수 전 필수:** A1.1–A1.4 통과 + 잔존활성 문헌 표에 face 위치 1차 기입(UNKNOWN 명시 허용).

### C1. 설계 성공 (계산 only)
| ID | 기준 | 수치 (잠정, 자문 후 확정) |
|----|------|---------------------------|
| C1.1 | 인터페이스 잔기가 사전 패치 집합과 겹침 | ≥ 50% |
| C1.2 | 촉매 6.5Å 내 바인더 원자 | **0** |
| C1.3 | AF-Multimer/복합체 신뢰 | 사전 정한 pLDDT/PAE 컷 (자문 후) |
| C1.4 | 후보 압축 | 패치당 ≤ 10 |

### C2. 설계 “실패지만 학습”
- 예쁜 복합체이나 C1.2 위반 → **폐기**, 성공 아님  
- docking score만 높음 → F1 실패 모드  

### C3. 금지 성공 선언
- ipTM 높음 = rescue  
- RFdiffusion 대량 생성 = 진전

## D. Wet / 협업 파일럿 (랩 합류 후)

### D1. 최소 양성 (소분자 또는 바인더 공통 철학)
| ID | 지표 | 잠정 컷 | 비고 |
|----|------|---------|------|
| D1.1 | ΔTm (DSF) | 방향성 + (≥ +1.5 °C 목표, 미달 시 “음성/경계” 기록 | 대리 지표 |
| D1.2 | 활성 | washout 또는 산성 pH에서 억제 완화 + 활성 비악화 | inhibitory PC |
| D1.3 | 발현/성숙 | 성숙 밴드 또는 EndoH 패턴 개선 | 트래픽 프록시 |
| D1.4 | 바인더 KD | 사전 범위 (예: < 1 μM, 자문 후) | 결합≠구제 |

**전체 성공:** D1.1 또는 D1.3 중 하나 **그리고** D1.2 비악화.  
**구제(rescue) 단어:** D1.2 + D1.3 동시 충족 전 사용 금지.

### D2. 중단 기준
- 반복 3회 음성 + 기술 문제 배제  
- 윤리/안전 이슈  
- claim 인플레이션 압력
