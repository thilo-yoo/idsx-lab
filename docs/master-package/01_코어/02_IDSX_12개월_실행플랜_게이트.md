## 0. 성공의 정의 (플랜 전체를 관통하는 KPI)

| 층 | 90일 성공 | 12개월 성공 |
|----|-----------|-------------|
| 문서 | amenability 맵 + 강도 맵이 자문 1회 이상 critique 반영 | 분류 보고서/논문 초고 + 공개 데이터 정리 |
| 계산 | S152I 등 ΔΔG 교차 1세트 + 해석 문서 | 선택 트랙 1개의 설계 파일럿 산출물 |
| 생물 | TIER1 잔존활성 문헌표 (UNKNOWN 명시 포함) | 최소 1종의 wet 또는 협업 프로토콜 합의 |

**실패로 치지 않는 것:** 치료 바인더 미완성, RFdiffusion 미실행, 논문 게재 실패.  
**실패로 치는 것:** non-claim 위반, Track 혼동 포장, 게이트 없이 대량 설계, 자문 없이 컷오프 임의 변경 후 확정 행세.

## 1. 모달리티 트랙 정의 (계획서 용어 고정)

### Track A — Amenability / 구조 모듈 / 표면 안정화

- **대상 예:** Ser152, Asp308, Tyr108 등 TIER1 코어·표면 불안정 + 촉매 이격
- **개입:** 인접 표면 패치에 대한 알로스테릭 소분자 또는 단백질 바인더 (후속)
- **1편 주축 후보**
- **성공 사전등록 예:** 성숙 밴드↑ 또는 잔존 활성↑ 또는 Tm↑ 중 **하나 이상**, 활성부위 비간섭

### Track B — Neo-pocket / 고ΔΔG 노출형 소분자 클램프

- **대상 예:** C422F/Y 등 CTSS 상위
- **개입:** 변이로 노출된 소수성 틈에 대한 클램프형 소분자 가설
- **주의:** 이황화 파괴형은 amenability Hard EXCLUDE와 **충돌 가능** → 별도 가설 문서로 관리
- **성공 사전등록 예:** 도킹만이 아니라 안정화 대리지표 + (가능 시) 활성 회복; EXCLUDE 논리와의 화해 문단 필수

### Track C — 비구제 / 주의

- null, 촉매 kill, 과도 인접(Arg88류)  
- 논문에서 **명시적 배제**가 기여

**규칙:** 발표·메일·포털에서 Track을 섞어 “최종 타겟 하나”로 강제 통합하지 않는다. 통합이 필요하면 **새 문서에 화해 논리**를 쓰고 게이트를 통과시킨다.

## 2. 90일 계획 (주 단위)

### Week 1–2 · 재료 정합 (D0–14)

| 작업 | 산출물 | 완료 기준 |
|------|--------|-----------|
| Ser152 계열 HGVS/ClinVar_id 중복 정리 | 정리 시트 1 | 중복 행 0 또는 명시 |
| 워딩 통일 | 용어 1페이지 | Hotspot/Epitope/Competence 정의 고정 |
| Track A/B 경계 문단 | 연대기 §2 Phase6 참조 | 포털 synthesis에 반영 |
| 자문 송부 | 본 패키지 + 질문지 | 발송 기록 |

**금지:** 새 대규모 계산 시작.

### Week 3–6 · 숫자 층 (D15–45)

| 작업 | 산출물 | 완료 기준 |
|------|--------|-----------|
| FoldX 또는 Rosetta ΔΔG | WT vs S152I (+S152R/N) | 방법·버전·명령 재현 기록 |
| 대조 변이 | Arg88 또는 표면 benign 1 | 같은 파이프라인 |
| (선택) AF template-off | 1 job | 기존 template-on과 비교 표 |
| C422 amenability 재검토 | 1페이지 메모 | EXCLUDE 규칙과 충돌 여부 명시 |

**판정:**

- ΔΔG가 불안정 방향이면 모듈 가설 **강화**  
- 애매하면 MD/실험 비중 상향, 설계 연기

### Week 7–10 · 생물 전제 (D46–70)

| 작업 | 산출물 | 완료 기준 |
|------|--------|-----------|
| TIER1 잔존활성·표현형 문헌 | 표 (PMID, 값, 세포형, 한계) | 상위 강조 위치부터 |
| UNKNOWN 정책 | 표기 규칙 | 채울 수 없음 ≠ 0 |
| IDS PC 문헌 재독 | Hoshina 등 노트 | 암브록솔 비중 vs IDS PC 비중 결정 |

### Week 11–13 · 설계 전 게이트 (D71–90)

| 작업 | 산출물 | 완료 기준 |
|------|--------|-----------|
| 표면 패치 2–3개 확정 | 5FQL 뷰 스냅샷 + 잔기 리스트 | Ser152 1패치 포함 권장 |
| 성공 기준 사전등록 | 1페이지 | 측정 항목·양성 기준·중단 기준 |
| 트랙 선택 | A 또는 B 중 **하나** 파일럿 | 다른 트랙은 문서만 유지 |
| 논문 목차 | IMRaD 초안 | claim/non-claim 박제 |

**90일 게이트 Go:** 위 표 4종 완성 + 자문 피드백 1회 이상 반영.  
**No-Go:** 파일럿 설계 착수 금지, 문헌·ΔΔG 보강 루프.

## 3. 12개월 분기 로드맵

### Q1 — 지도 완성

- amenability 맵 고정 (위치 ≤20 본문 강조)
- ΔΔG 패널 + 잔존활성 문헌표
- 자문 1–2회  
**Gate Q1:** 본문 강조 위치 수 합의, 6.5Å 지위(유지/수정) 합의

### Q2 — 파일럿 1트랙

- 패치 1개 확정
- 소분자 **또는** 바인더 중 하나
- 계산 파이프만 또는 협업 assay 설계  
**Gate Q2:** 성공 기준 사전등록 문서 서명(본인·자문)

### Q3 — 검증 접촉

- 발현 / DSF / SPR 중 가능한 최소 실험 **또는** 공개 데이터 재분석
- Methods 확정  
**Gate Q3:** non-claim 내부 리뷰 0건 위반

### Q4 — 산출·공개

- 분류 논문/보고서 초고
- 데이터·코드 정리, idsx-lab 업데이트
- 랩 합류용 포트폴리오  
**Gate Q4:** 외부 1명 critique 반영 로그
