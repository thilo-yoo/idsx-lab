# PFI — Process Fitness Index 워크시트 (v0)

**정의:** 위치/변이가 “공정 효율 항(η)을 건드리는 결함 + 개입 가능성” 관점에서  
얼마나 **파일럿 검토 가치가 있는지** 나타내는 내부 지수.  
**CTSS와 분리.** CTSS는 Track B 전용.

\[
\mathrm{PFI} = 100 \times \big(0.35\,\tilde s + 0.25\,\tilde n + 0.25\,\tilde d + 0.15\,\tilde e\big) - 40\,P_{\mathrm{ex}}
\]

사전등록 가중치 (결과 보고 바꾸지 말 것; 바꾸면 v1):

| 기호 | 의미 | 정규화 |
|------|------|--------|
| \(\tilde s\) | amenability_score / 10 | 0–1 |
| \(\tilde n\) | network/module proxy | 0–1 아래 표 |
| \(\tilde d\) | min(dist/15, 1) | 0–1 (15Å에서 포화) |
| \(\tilde e\) | min(epitope_count/12, 1) | 0–1 |
| \(P_{\mathrm{ex}}\) | 1 if EXCLUDE or CAUTION_for_binder else 0 | |

### network proxy \(\tilde n\) 임시 규칙
| 조건 | \(\tilde n\) |
|------|--------------|
| 명시적 모듈 카드 (152, 308 등) | 1.0 |
| core RSA&lt;10 + TIER1 | 0.7 |
| surface TIER1 | 0.5 |
| else | 0.3 |

## 사용 규칙

1. PFI는 **내부 우선순위**이지 논문 메인 claim 숫자가 아님 (원하면 Suppl).  
2. residual YES가 나오면 가산점 규칙은 **v1에서 사전등록 후**만.  
3. Track B 후보는 PFI 대신 `Druggability` 표.  
4. 발표 시 PFI 랭킹만 보여 주고 구조를 숨기지 말 것.
